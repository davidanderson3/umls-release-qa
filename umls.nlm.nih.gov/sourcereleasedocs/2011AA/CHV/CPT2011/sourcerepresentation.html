Notes

CPT data files are submitted by the American Medical Association (AMA). The files processed for the UMLS Metathesaurus are available for licensing at the AMA Bookstore.

The information provided here applies explicitly to the English version of CPT. The Spanish translation of CPT (CPTSP) was last updated in 2001, and much of the information on this page is not applicable.

Summary of Changes:

  (return to top)

No changes have been made to the original data file format. For 2011, there are no longer any non-ascii characters in the .txt files, so there are no longer atoms with TTY = "MTH_PT" or "MTH_SY," and no relationship attributes of  "mth_plain_text_form_of/mth_has_plain_text_form".

Notes: The current license agreement for including CPT in the Metathesaurus includes the above files. The AMA has recently begun distributing the CPT® Developer's Toolkit which includes additional data such as a hierarchy, relationships and additional attributes. The NLM is currently evaluating CPTDTK, and working with the AMA to determine whether it would be appropriate to license the DTK content for the Metathesaurus. For more information on CPTDTK, please see the AMA website.

The standard CPT data files distributed by the AMA do not include a hierarchy. The NLM uses the printed book and other resources to generate a hierarchy during Metathesaurus source processing. This hierarchy has not been reviewed by the AMA. In addition, the NLM generates relationships between CPT codes based on string parsing. The hierarchy and relalationships are attributed to SAB = "MTHCH".

Not included: All information from the CPT data files is included in the UMLS representation.
* Certain fields and files may not be directly processed because they contain redundant data.

Modifiers. This file is not a set format and must be manual edited before Metathesaurus source processing.

# Field Name Description Representation
1
 CPT Code 2-digit alpha-numeric code MRCONSO.CODE
MRCONSO.SCUI
(TTY = "MP")
2
 CPT Modifier Description (generally, after the table character) Mixed-case alpha-numeric string MRCONSO.STR
MRSAT.ATN = "CPF"
The MP atom is generally created by extracting the text before the first ":". Material appearing after the first ":" is included in the "CPF" attribute.
