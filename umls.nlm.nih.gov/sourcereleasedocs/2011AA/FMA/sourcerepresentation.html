Term Types:

RELAs:
articulates_with
adheres_to/has_adherent
has_part/part_of

Hierarchical RELA (previously null):
isa

ANATOMICAL_COORDINATE (replaces COORDINATE)
FREESURFER_TERM
NEUROLEX_ID
RADLEX_ID
TALAIRACH
TALAIRACH_ID

The following data types were deleted in FMA3_1:

RELAs:
has_homonym/homonym_for

Attributes:
COORDINATE

The following RELAs were updated due to a spelling error.  This correction resulted in new RUI assignments for existing relationships:
eipthelial_cell_shape_of --> epithelial_cell_shape_of
has_eipthelial_cell_shape --> has_epithelial_cell_shape

Source-Provided Files: Summary

  (return to top)

The FMA distribution includes the following. These files, along with additional information can be accessed at http://sig.biostr.washington.edu/projects/fm

Data files:

File Name
fma_3.1_frames_db.sql
fma_3.1_frames_db.pprj

To process FMA, fma_3.1_frames_db.sql was loaded into a MySQL database for Protegè.  Protegè's export function was then used to write "classes.fma.pont" and "instances.fma.pins."  The .pins and .pont files were processed for the Metathesaurus.

Not included: 

All information in the file FMA, fma_3.1_frames_db.sql is processed for the Metathesaurus.
