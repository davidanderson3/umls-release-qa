Notes

MEDCIN data files are submitted by Medicomp Systems.

Summary of Changes

No changes have been made to the format of submitted MEDCIN data files.  The following changes have been made to Metathesaurus processing:

  • Hierarchy
    • Now processed at the SCUI level:.  PAR/CHD rels in MRREL.RRF are connected to SCUI's rather than AUI's.  
    • Additional concepts included in hierarchy;  AUI of FN used if no PT:  A small number of MEDCIN concepts do not have PT atoms.  Previously, no hierarchy was created for these concepts.  Now, if there is no PT, the AUI of the FN will be used in MRHIER.RRF and MRREL.RRF.  There are still a handful of concepts which are not included in the hierarchy;  this is due to problems with their NODEKEY values in the original data.
  • MEDCIN to SNOMEDCT relationships:  The following new linkage concepts are used:

Hierarchy

# Field Name Description Representation
1 MEDCINID MEDCIN concept ID MRHIER.CUI populated by the Metathesaurus CUI corresponding to MEDCINID.
2 NODEKEY Hierarchical identifier Used to construct the hierarchical tree position, MRHIER.PTR, for this concept. The "NODEKEY" value is a series of two-character tokens (some of which include a space character), e.g., " CDA D P" breaks down as " C", "DA", " D", and "P", and removing the last token (yielding " CDA D", in this example) produces the "NODEKEY" value of this concept's hierarchical parent.

MEDCIN-SNOMEDCT mappings.

# Field Name Description Representation
1 MEDCINID MEDCIN concept ID MRREL.CUI1 populated by the Metathesaurus CUI corresponding to MEDCINID
2 RELATIONSHIPTYPE The SNOMEDCT CONCEPTID indicating the type of this relationship Mapped to corresponding values in MRREL.REL and MRREL.RELA. E.g., the "RELATIONSHIPTYPE" 123005000, which is the SNOMEDCT CONCEPTID for "Part of (attribute)", is mapped to REL = "RN" (narrower relationship) and RELA = "part_of".

If RELATIONSHIPTYPE = "168666000" (same_as), MRMAP.RRF and MRSMAP entries are also created.
3 SCT_CONCEPTID The SNOMEDCT CONCEPTID of the concept having the specified type of relationship to the concept indicated by MEDCINID MRREL.CUI2 populated by the Metathesaurus CUI corresponding to SCT_CONCEPTID
4 PREF_FLAG_ID 1=preferred, 0=non-preferred
Indicates the preferred mapping.  In some cases,there may be more than one mapping with value = "1".
 MRSAT.ATN = "PREF_FLAG_ID" (STYPE = "RUI")
