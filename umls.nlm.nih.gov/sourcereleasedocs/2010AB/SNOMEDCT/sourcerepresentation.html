SNOMEDCT data files are submitted by IHTSDO.

Summary of Changes

  (return to top)

There have been no changes to the SNOMEDCT format.  Metathesaurus processing, the follwing changes have occurred for 2010AB:

  •   Value of "MTH_UMLSMAPSETSEPARATOR" attribute correctly set to "AND" which is used in MRMAP.RRF and MRSMAP.RRF

In the future, the new SNOMEDCT RF2 format will be processed, and this document will be updated to reflect the changes.

Relationships

# Field Name Description Representation
1 RELATIONSHIPID SNOMEDCT Relationship ID MRREL.SRUI
2 CONCEPTID1 SNOMEDCT Concept ID of relationship's source concept MRREL.CUI2 populated by the Metathesaurus CUI corresponding to CONCEPTID1
3 RELATIONSHIPTYPE SNOMEDCT Concept ID of concept representing the relationship between the related concepts A relationship-name mapping is used to set MRREL.REL and MRREL.RELA based on this value.

These mappings are expressed as attributes with MRSAT.ATN="UMLSREL" and MRSAT.ATN="UMLSRELA", specifying the REL and RELA values used for eacah RELATIONSHIPTYPE.   For historical reasons, however, the UMLSREL attribute values "RT" and "NT" are used for REL values in MRREL.RRF of "RO" and "RN", respectively
4 CONCEPTID2 SNOMEDCT Concept ID of relationship's target concept MRREL.CUI1 populated by the Metathesaurus CUI corresponding to CONCEPTID2
5 CHARACTERISTICTYPE Indicates whether this relationship is a defining (0), qualifying (1), historical (2), or other additional (3) characteristic of the source concept. MRSAT.ATN="CHARACTERISTICTYPE" where MRSAT.METAUI is the MRREL.RUI assigned where MRREL.SRUI = this relationship's RELATIONSHIPID

Also used to create MRSAT.ATN="AQ" where CHARACTERISTICTYPE = 1 (qualifier) and REFINABILITY = 2 (mandatory)
6 REFINABILITY Indicates whether the target concept must (2), may (1), or may not (0) be refined when using this relationship as a clinical data entry template. MRSAT.ATN="REFINABILITY" where MRSAT.METAUI is the MRREL.RUI assigned where MRREL.SRUI = this relationship's RELATIONSHIPID

Also used to create MRSAT.ATN="AQ" where CHARACTERISTICTYPE = 1 (qualifier) and REFINABILITY = 2 (mandatory)
7 RELATIONSHIPGROUP An integer indicating whether this relationship is (if non-zero) or is not (if 0) associated with other relationships. When non-zero, all relationships having the same CONCEPTID1 and RELATIONSHIPGROUP values are associated. MRREL.RG

ICD9 Cross Map Sets

# Field Name Description Representation
1 MAPSETID SNOMEDCT Cross Map Set ID MRCONSO.SCUI and CODE for an atom with SAB="SNOMEDCT", TTY="XM";
for attributes, MRSAT.CUI populated by the Metathesaurus CUI corresponding to the XM atom for this MAPSETID
2 MAPSETNAME A name describing this Cross Map Set MRSAT.ATN="MAPSETNAME"
3 MAPSETTYPE Indicates the type of maps in this set:
0 => Unspecified
1 => Single (all one-to-one maps)
2 => Multiple (some one-to-many maps, but no maps to a list of choices)
3 => Choices (some maps specify a choice of multiple one-to-one maps, but no one-to-many maps)
4 => Flexible (some one-to-many maps, and some maps specifying choices)		 MRSAT.ATN="MAPSETTYPE"
4 MAPSETSCHEMEID Standard ID for the Target Scheme the map set belongs to MRSAT.ATN="MAPSETSCHEMEID"
5 MAPSETSCHEMENAME Full name of the map set Target Scheme MRSAT.ATN="MAPSETSCHEMENAME"
6 MAPSETSCHEMEVERSION Version number of the Target Scheme, as published by the issuing organization MRSAT.ATN="MAPSETSCHEMEVERSION"
7 MAPSETREALMID Identifier of the realm in which the map set is applicable MRSAT.ATN="MAPSETREALMID"
8 MAPSETSEPARATOR The character used as a separator when a map target includes multiple codes MRSAT.ATV set to XML entity code for this character with MRSAT.ATN="MAPSETSEPARATORCODE"
9 MAPSETRULETYPE Indicates the types of rules used in this map set's cross maps and cross map targets MRSAT.ATN="MAPSETRULETYPE"

ICD9 Cross Map Targets

# Field Name Description Representation
1 TARGETID SNOMEDCT ID for this Cross Map Target MRMAP.TOID, MRMAP.TOSID; MRMAP.TOTYPE set to "BOOLEAN_EXPRESSION_SDUI"
2 TARGETSCHEMEID Standard ID for the Target Scheme this target belongs to Checked for matching the MAPSETSCHEMEID of the Cross Map Set for ICD9
3 TARGETCODES A code or list of codes from the Target Scheme that may be mapped to from SNOMEDCT concepts MRMAP.TOEXPR (computed from TARGETCODES by replacing the MAPSETSEPARATOR character with " AND ")
4 TARGETRULE A machine-processable expression of rules that determine when this target applies MRMAP.TORULE
5 TARGETADVICE Textual advice describing when this target applies (this field is currently always blank)

ICD9 Cross Maps

# Field Name Description Representation
1 MAPSETID SNOMEDCT Cross Map Set ID MRMAP.MAPSETCUI populated by the Metathesaurus CUI corresponding to MAPSETID
2 MAPCONCEPTID SNOMEDCT ID of the mapped concept MRMAP.FROMID, MRMAP.FROMEXPR
3 MAPOPTION An integer distinguishing between alternative mappings for a concept (this field is currently always 0)
4 MAPPRIORITY An integer indicating the suggested order for presenting a list of options for mapping a concept MRMAP.MAPRANK
5 MAPTARGETID The SNOMEDCT ID of a Cross Map Target that this concept may be mapped to Matched to the TARGETID column of the Cross Map Targets table to find the data for the "To" side of the mapping in MRMAP
6 MAPRULE A machine processable expression of rules to determine when this Cross Map applies MRMAP.MAPRULE
7 MAPADVICE Textual advice to support a person deciding when this Cross Map applies (this field is currently always blank)
