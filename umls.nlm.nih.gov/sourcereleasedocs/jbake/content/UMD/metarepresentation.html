The UMLS uses an xml version of UMDNS. The file used for 2021AA is "NLM_UMLS_UMDNS2020_version7.xml"

Note: Retired UMD concepts are not included in the UMLS Metathesaurus."

Identifiers:

UMDNS includes both concepts, i.e. collections of synonymous atoms, and descriptors, i.e. collections of concepts that are considered to be closely related (but not necessarily synonymous). Therefore, atoms with TTY = "SY" are assigned the same Source-asserted Concept Identifer (SCUI), and Source-asserted Descriptor Identifier (SDUI) as their PT. However, atoms with TTY = "ET" are considered to be in a different concept from their "PT" (different SCUI) but members of the same descriptor (same SDUI). Identifiers are assigned as follows:

  • CODE:
    For termtypes "PT", "ET", "RT", "HT": populated by the "CODE_IN_SOURCE" property.
    For termtype "SY": populated by the "CODE_IN_SOURCE" value of the "PT"
  • SAUI:
    Not Applicable
  • SCUI:
    Same as "CODE"
  • SDUI:
    For termtypes "PT", "SY", "ET": CODE of "PT"
    For termtype "HT": CODE of "HT"
    For termtype "RT": CODE of "RT" (An "RT" may have multiple "PT"s, so we can't use code of "PT")

Termtypes are assigned based on the value of the "DEVICE_TYPE" property. "PT" atoms are created where the "DEVICE_TYPE" is one of:
  • CATEGORY EXPANDED BELOW entry with SEE ALSO references
  • CATEGORY EXPANDED BELOW entry
  • Device has SEE ALSO references
  • Main Device Term
  • NIR entry

STR = value of "conceptDef<name>" property
SCUI = value of "CODE_IN_SOURCE" property
SDUI = value of "CODE_IN_SOURCE" property

Termtypes are assigned based on the value of the "DEVICE_TYPE" property. "HT"s are created where the "DEVICE_TYPE" is:
  • Hierarchical Term
RT CODE = value of "CODE_IN_SOURCE" property
STR = value of "conceptDef<name>" property
SCUI = value of "CODE_IN_SOURCE" property
SDUI = value of "CODE_IN_SOURCE" property

Termtypes are assigned based on the value of the "DEVICE_TYPE" property. "RT"s are created where the "DEVICE_TYPE" is:
  • Device is a USE MORE SPECIFIC TERM entry.
SY CODE = value of "CODE_IN_SOURCE" property of associated "PT"
STR = value of the "SYNONYM" property
SCUI = value of "CODE_IN_SOURCE" property of associated "PT"

ET CODE = value of "CODE_IN_SOURCE" property

SCUI = value of "CODE_IN_SOURCE" property
SDUI = value of "CODE_IN_SOURCE" property of associated "PT"

Termtypes are assigned based on the value of the "DEVICE_TYPE" property. ETs are created where the "DEVICE_TYPE" is:
  • Device is a SEE reference.

Attributes (MRSAT):

Attribute Name Origin of ATV

  • CATEGORY EXPANDED BELOW entry with SEE ALSO references

  • Device is a SEE reference
  • Device is a USE MORE SPECIFIC TERM entry

SYCODE The value of the "Synonym_Code" qualifier, if present. Attached to AUI.

Definitions (MRDEF):

ORIGIN
The value of the "DEFINITION" property, where value of the "Definition _Status" qualifier = "Releasable." Definitions with other qualifier values are excluded.
