Summary of Changes:

No changes have been made to the original data files or to representation of UMDNS.

Source Provided Files:

The UMLS uses an xml version of UMDNS which is created with the TDE tool from Apelon, Inc. The file used for 2013AA is "NLM_20121130_v4.xml" (the defined view). (http://apelon.com/dtd/tde/ontylog.dtd)

Identifiers:

UMDNS includes both “concepts,” i.e. collections of synonymous atoms, and “descriptors,” i.e. collections of concepts that are considered to be closely related (but not necessarily synonymous). Therefore, atoms with TTY = "SY" are assigned the same Source-asserted Concept Identifer (SCUI), and Source-asserted Descriptor Identifier (SDUI) as their PT. However, atoms with TTY = "ET" are considered to be in a different concept from their "PT" (different SCUI) but members of the same descriptor (same SDUI). Identifiers are assigned as follows:

  • CODE:
    For termtypes "PT", "ET", "RT", "HT": populated by the "CODE_IN_SOURCE" property.
    For termtype "SY": populated by the "CODE_IN_SOURCE" value of the "PT"
  • SAUI:
    Not Applicable
  • SCUI:
    Same as "CODE"
  • SDUI:
    For termtypes "PT", "SY", "ET": CODE of "PT"
    For termtype "HT": CODE of "HT"
    For termtype "RT": CODE of "RT" (An "RT" may have multiple "PT"s, so we can't use code of "PT")

DEVTYPE The value of the "DEVICE_TYPE" property. One of the following:
  • CATEGORY EXPANDED BELOW entry with SEE ALSO references

  • Device has SEE ALSO references

Note: In the original data, some synonyms have a qualifier of "Synonym_Device_Type," and some are assigned a blank or incorrect DEVICE_TYPE.
For consistency, in the Metathesaurus, all SY atoms are defaulted to "Device is a SEE reference." DEVTYPE is attached to AUI for SY atoms; for all other atoms, it is attached to CODE.
SYCODE The value of the "Synonym_Code" qualifier, if present. Attached to AUI.
